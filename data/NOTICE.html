Importing Data in the Demo

Demo parameters.

In the init.xlsx first tab you will find setup parameters for your pim demo. Each parameters are described by an excel comment.

Importing Products.

You can add as many product import files as needed in this directory. The name of the files should start by products and they should have the xlsx extension.

You can use the columns you want inside the files, and as many worksheets as needed. Attributes will be created for each of your columns.

The following special columns can be used:

sku
Use this column to specify the identifier of the products. If the column is not present, or no SKU is read, an identifier will be automatically generated.
family
Use this column to specify the name of the product family. If the column is not present, or no value is given, the name of the worksheet will be used. All the attributes of the product will be added to its family.
category1, category2... category6
Use these columns to specify the name of the categories in which the product should be classified. The categories will be created accordingly.
For example, if you have "T-shirt" in category1 and "Long sleeves" in category2, the product will be in Master>T-shirt>Long sleeves.

For all special columns, you must use the exact column header, in lowercase.

You can also use the following attributes in your data:

Name
Will be used for the label of products, and in the rules engine. (You may also use Variation name if you already used name in the product model)
Description
Will be used for the rules engine.
Supplier
Will be used to populate the suppliers database and to create supplier user accounts. For example, if you add "Zaro" in the supplier column, you will be able to connect to the pim with a zaro:zaro account.

Importing Product Models.

You can add as many product model import files as needed in this directory. The name of the files should start by product_models and they should have the xlsx extension.

You can use the columns you want inside the files, and as many worksheets as needed. Attributes will be created for each of your columns.

The following special column must be used:

code
The unique code of the product model. Can be use in the parent column of the product.xlsx file.
family_variant
The family variant of the product model. Be careful each product link to a product model must have the same family.
variant-axes_1
The attribute(s) that should be use has axes. You should declare it in the init.xlsx.

The following special columns can be used:

parent
If the product model is a second level product model. In that case you must specify variant-axes_2

For all special columns, you must use the exact column header, in lowercase.

You can also use the following attributes in your data:

Name
Will be used for the label of products.
image_1
Will be used for the main product image. (displayed in the grid)

Importing Reference Entity Records.

You can add as many reference entity record import files as needed in this directory. The name of the files should start by reference_entit and they should have the xlsx extension.

You can use the columns you want inside the files, and as many worksheets as needed. Attributes will be created for each of your columns.

The following special column must be used:

reference_entity
The code or label of the reference entity.
label
The label of the record. A code will be automatically generated from the label.

The following special columns can be used:

image
Will be used for the main record image. (displayed in the grid).

Customizing the attributes.

You can change the type of the attributes, as well as their order, and their attribute groups in the init.xlsx file of the folder. To change the properties an attribute, copy its name in the label column, and customize the attribute's data. The following rules must be followed:

  • If you select the Metric type, you will have to specify metric_family and default_metric_unit
  • If you select the Image attribute type or the File attribute type, you will have to specify the allowed_extensions.

Importing Images.

To import images, proceed as follows:

  • Save your images in the data/images folder.
  • Inside your product import files, set the relative path of the images for all images. For example, if you have a image.jpeg file in your images folder, you will need to have a value of images/image.jpeg
  • Set the type of the image attributes to Image in the init.xlsx file or use Image 1, Image 2 or Image 3 column name.
  • If you have image links in you data file you can set download_images to True in the init.xlsx file in order to automatically download them.

Asset collections will be created from the images of your products.
